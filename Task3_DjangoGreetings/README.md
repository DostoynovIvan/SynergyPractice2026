# Задача 3. Django-приложение «Персонализированное приветствие»

## Описание
Веб-приложение на Django, которое позволяет пользователю ввести своё имя, 
сохраняет его в базе данных и выводит персонализированное приветствие на главной странице. 
Реализована обработка ошибок (пустое поле), защита от CSRF, стилизация с помощью CSS.

## Функциональность
- Форма ввода имени с кнопкой «Отправить».
- Сохранение имени в базу данных (модель `UserName` с полем `name`).
- Отображение приветствия после успешной отправки: «Привет, {имя}!».
- Валидация: если поле пустое, форма возвращает ошибку.
- CSRF-защита (встроенная в Django).
- Стилизованный интерфейс (градиенты, тени, адаптивность).

## Технологии
- Python 3.x
- Django 4.x / 5.x
- SQLite (база данных по умолчанию)
- HTML5, CSS3

## Инструкция по запуску

### 1. Перейти в папку проекта
Откройте терминал и выполните:
```bash
cd Task3_DjangoGreetings

### 2. Создать и активировать виртуальное окружение (рекомендуется)

Windows:
bash

python -m venv venv
venv\Scripts\activate

macOS/Linux:
bash

python3 -m venv venv
source venv/bin/activate

### 3. Установить зависимости
bash

pip install -r requirements.txt

(Если файл requirements.txt отсутствует, установите Django вручную: pip install django)

### 4. Применить миграции (создать базу данных)
bash

python manage.py migrate

### 5. Запустить сервер разработки
bash

python manage.py runserver

### 6. Открыть в браузере

Перейдите по адресу http://127.0.0.1:8000/

Структура проекта (основные файлы)

    manage.py – точка входа для команд Django.

    greetings_project/ – папка с настройками проекта.

    greetings/ – основное приложение:

        models.py – модель UserName.

        forms.py – форма NameForm на основе модели.

        views.py – логика обработки запросов.

        urls.py – маршруты приложения.

        templates/greetings/index.html – шаблон страницы.

        static/greetings/style.css – стили.

    db.sqlite3 – файл базы данных (создаётся после миграций).

Примечания

    Убедитесь, что виртуальное окружение активировано перед запуском команд.

    Если возникают ошибки, проверьте версию Python и Django.

    Для остановки сервера нажмите Ctrl+C в терминале.
